## 1. 系统概述

本系统是一套**面向 B2B 外贸企业**的轻量级网站框架，采用无第三方依赖的纯 PHP 实现。系统包含：

- **前台**：产品展示、博客文章、成功案例、询盘表单、联系表单
- **后台**：内容管理、设置配置、媒体管理、询盘管理、菜单/轮播图编辑

**核心设计原则：**
- 零框架依赖，PHP 内置功能全覆盖
- SQLite 文件数据库，无需单独安装数据库服务
- PSR-4 命名空间自动加载，代码组织清晰
- 版本化数据库迁移，安装部署自动完成
- 主题系统支持多主题切换，区块系统支持可视化内容编辑

## 2. 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 运行环境 | PHP 8.1+ | 需要 `SQLite3`、`mbstring`、`fileinfo` 扩展 |
| 数据库 | SQLite3 | 文件数据库，存储于 `storage/site.db` |
| Web 服务器 | Apache / Nginx | 需要 URL 重写（mod_rewrite / try_files） |
| 前端 CSS | Tailwind CSS CDN（JIT 模式） | 通过 CDN 引入，无需构建工具 |
| 前端图标 | Font Awesome 6.4.0 CDN | 图标库 |
| 前端轮播 | Swiper.js 11 CDN | 仅在需要页面引入 |
| 富文本编辑器 | Jodit | 后台文章/产品内容编辑 |
| 拖拽排序 | SortableJS | 菜单排序 |

## 5. 请求生命周期

以访问 `GET /product/steel-pipe` 为例：

```
1. Apache .htaccess 规则将请求转发到 index.php
   └── RewriteRule ^ index.php [L]

2. index.php 执行：
   ├── 定义常量 APP_ROOT、APP_BASE_PATH
   ├── 开启 Session、设置时区/编码
   ├── 配置错误处理（DEBUG 模式开关）
   ├── 注册 PSR-4 自动加载器（App\\ → app/）
   ├── require app/Helpers/Helpers.php（全局函数）
   ├── Database::getInstance()
   │   └── Migrator::runAllPending()（首次或有新迁移时执行）
   └── Router::run()

3. Router 匹配路由：
   └── GET /product/:slug → [SiteController::class, 'productDetail']

4. SiteController::productDetail('steel-pipe')：
   ├── BaseController::__construct() → loadSiteData()（加载所有 settings）
   ├── Product::getBySlug('steel-pipe')（模型查询）
   ├── 检查产品状态 → 不存在则 notFound()
   ├── 加载相关产品、阶梯价格、面包屑等数据
   └── $this->renderSite('product_detail', [...])

5. BaseController::renderSite() → Controller::render()：
   ├── 加载主题 functions.php
   ├── echo themes/default/header.php（<head> + 导航）
   ├── echo themes/default/product_detail.php（页面内容）
   └── echo themes/default/footer.php（页脚）

6. 输出 HTML → 返回客户端
```

## 6. 核心模块详解

### 6.1 入口文件 `index.php`

**职责：** 应用启动、全局配置、自动加载、路由分发。

**关键常量：**

| 常量 | 说明 | 示例值 |
|------|------|--------|
| `APP_ROOT` | 项目根目录绝对路径 | `/var/www/html/mysite` |
| `APP_BASE_PATH` | 二级目录部署前缀 | `/myapp`（根目录部署时为空字符串） |
| `APP_DEBUG` | 是否开启调试模式 | `true` / `false` |
| `APP_ENTRY_POINT` | 入口标记，防止直接访问 app 文件 | `true` |

**PSR-4 自动加载规则：**
```
命名空间前缀 App\  →  目录 app/
App\Core\Router   →  app/Core/Router.php
App\Models\Product →  app/Models/Product.php
```

**调试模式控制：**

在项目根目录创建 `.env` 文件，写入：
```
APP_DEBUG=true
```
开启后：控制台显示错误详情；关闭时：错误写入 `uploads/logs/error.log`。

### 6.2 路由系统 `Router`

**文件：** `app/Core/Router.php`

**工作原理：**

```php
// 注册路由（在 app/routes.php 中）
$router->add('GET',  '/product/:slug', [SiteController::class, 'productDetail']);
$router->add('POST', '/contact',       [SiteController::class, 'contact']);

// :slug 自动转换为正则 (?P<slug>[a-z0-9\-]+)
// 匹配后 slug 作为参数传入控制器方法
```

**路由注册规范（`app/routes.php`）：**

| URL 前缀 | 控制器 | 说明 |
|----------|--------|------|
| `/` | `SiteController` | 前台所有页面 |
| `/admin/*` | `AdminController` | 后台所有功能 |
| `/robots.txt` | `SiteController::robots` | 动态生成 robots |
| `/sitemap.xml` | `SiteController::sitemap` | 动态生成 sitemap |

**路径参数规则：**
- 仅支持小写字母、数字、连字符：`[a-z0-9\-]+`
- 每个段落用 `:paramName` 声明，如 `/product/:slug`

### 6.3 数据库层 `Database`

**文件：** `app/Core/Database.php`

**类型：** 单例模式（Singleton）

```php
$db = Database::getInstance();  // 全局唯一连接
```

**SQLite 配置：**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 文件路径 | `storage/site.db` | 禁止 Web 访问 |
| 日志模式 | WAL（Write-Ahead Logging） | 支持并发读写 |
| 外键约束 | 启用 | `PRAGMA foreign_keys = ON` |
| 忙超时 | 5000ms | 避免并发锁死 |
| 文件权限 | `0640` | 仅 Web 服务器用户可读写 |

**连接建立流程：**

```
Database::getInstance()
├── 检查 $instance 是否为 null
├── 构建 dbFile 路径：APP_ROOT/storage/site.db
├── 若文件不存在（首次）：
│   ├── 创建 storage/ 目录
│   └── 写入 storage/.htaccess 防止直接访问
├── new SQLite3($dbFile)
├── 配置 PRAGMA 参数
└── Migrator::runAllPending()（自动执行待处理迁移）
```

### 6.4 迁移系统 `Migrator`

**文件：** `app/Core/Migrator.php`

**设计理念：** 每个迁移文件对应数据库的一次变更，按时间戳顺序执行，只执行一次，执行结果记录在 `migrations` 表中。

**迁移文件命名规范：**
```
{YYYYMMDDHHMMSS}_{描述}.php
例：20260410000003_create_products_table.php
```

**迁移文件结构（匿名类）：**
```php
<?php
declare(strict_types=1);

return new class {
    public function up(SQLite3 $db): void {
        // 正向：创建表、添加索引、插入默认数据
        $db->exec('CREATE TABLE IF NOT EXISTS ...');
    }
    
    public function down(SQLite3 $db): void {
        // 回滚：删除表（当前系统暂未实现自动回滚）
        $db->exec('DROP TABLE IF EXISTS ...');
    }
};
```

**执行流程：**
```
Migrator::runAllPending()
├── 读取 app/Migrations/*.php 文件列表
├── 按版本号（时间戳）排序
├── 查询 migrations 表获取已执行版本
├── 过滤出未执行的迁移
└── 逐个执行 up() 方法，记录到 migrations 表
```

| 20260410000001 | create_users_table | `users` 表 + 默认 admin 账户 |
| 20260410000002 | create_settings_table | `settings` 表 + 38 条默认配置 |
| 20260410000003 | create_products_table | `products` 表 |
| 20260410000004 | create_posts_table | `posts` 表（文章/案例/页面共用） |
| 20260410000005 | create_inquiries_table | `inquiries` 询盘表 |
| 20260410000006 | create_messages_table | `messages` 留言表 |
| 20260410000007 | create_product_categories_table | `product_categories` 分类表 |
| 20260410000008 | create_product_prices_table | `product_prices` 阶梯价格表 |
| 20260410000009 | create_media_files_table | `media_files` 媒体索引表 |
| 20260410000010 | create_update_logs_table | `update_logs` 更新日志表 |

### 6.5 控制器层

**继承链：**
```
Controller (abstract)          ← 渲染、JSON、重定向基础方法
    └── BaseController         ← 加载全局站点设置，renderSite()
            ├── SiteController ← 前台所有页面逻辑
            └── AdminController ← 后台所有管理功能
```

**`Controller` 核心方法：**

| 方法 | 说明 |
|------|------|
| `render(theme, view, data)` | 渲染前台页面（header + view + footer） |
| `json(data, status)` | 返回 JSON 响应并 exit |
| `redirect(url)` | HTTP 302 跳转 |

**`BaseController` 职责：**
- 构造时调用 `loadSiteData()` 一次性加载所有 `settings` 到 `$this->siteData`
- 提供 `renderSite(view, data)` 方法，将站点数据自动合并到视图变量
- 静态缓存 `$cachedSettings`，同一请求内只查询一次数据库

**`SiteController` 路由方法：**

| 方法 | 路由 | 说明 |
|------|------|------|
| `home()` | `GET /` | 首页 |
| `products()` | `GET /products` | 产品列表（支持分类筛选） |
| `productDetail(slug)` | `GET /product/:slug` | 产品详情 |
| `cases()` | `GET /cases` | 案例列表 |
| `caseDetail(slug)` | `GET /case/:slug` | 案例详情 |
| `blog()` | `GET /blog` | 博客列表 |
| `blogDetail(slug)` | `GET /blog/:slug` | 博客详情 |
| `pageDetail(slug)` | `GET /page/:slug` | 独立页面 |
| `about()` | `GET /about` | 关于我们 |
| `contact()` | `GET\|POST /contact` | 联系我们 |
| `inquiry()` | `POST /inquiry` | 产品询盘提交 |
| `robots()` | `GET /robots.txt` | 动态 robots.txt |
| `sitemap()` | `GET /sitemap.xml` | 动态 sitemap.xml |

### 6.6 模型层

**文件：** `app/Models/`

所有模型继承 `BaseModel`，通过 `Database::getInstance()` 获取连接。

**`BaseModel` 封装的查询方法：**

| 方法 | 说明 |
|------|------|
| `fetchAll(sql, params)` | 返回结果集数组 |
| `fetchOne(sql, params)` | 返回单行或 null |
| `getSqliteType(val)` | 自动推断 SQLITE3_INTEGER / TEXT / FLOAT / NULL |

**主要模型说明：**

| 模型 | 表 | 关键说明 |
|------|----|---------|

### 6.7 视图渲染

**前台渲染流程（`Controller::render`）：**

```
render('default', 'product_list', $data)
├── 1. 确定主题路径：themes/default/（不存在则回退到 themes/default/）
├── 2. 加载 themes/default/functions.php（主题专属辅助函数）

├── 5. 输出 themes/default/product_list.php（页面主体）
└── 6. 输出 themes/default/footer.php
```

| `$seo` | array | 当前页面 SEO（title、description、keywords） |

**后台视图（`app/views/admin/`）：**

后台视图直接 `include` 到 `AdminController` 中，使用布局文件 `app/views/admin/layout.php` 包裹。

## 11. 安全架构

### 多层防护

```
第一层：Apache .htaccess
  ├── RewriteRule ^storage/ - [F,L]       禁止访问数据库目录
  ├── RewriteRule \.(db|sqlite|env|log)$  禁止访问敏感文件
  ├── RewriteRule ^\.git/                 禁止访问版本控制目录
  └── Options -Indexes                    禁止目录列表

第二层：storage/.htaccess（自动生成）
  └── Deny from all                       双重保护数据库目录

第三层：index.php 安全响应头
  ├── X-Content-Type-Options: nosniff
  ├── X-Frame-Options: SAMEORIGIN
  ├── X-XSS-Protection: 1; mode=block
  └── Referrer-Policy: strict-origin-when-cross-origin

第四层：应用层
  ├── csrf_check()   所有 POST 请求验证 CSRF 令牌
  ├── h()            所有输出必须转义
  ├── AuthManager    后台路由守卫
  ├── 文件上传：     MIME 验证、大小限制、getimagesize 二次验证
  └── 数据库：       PreparedStatement 防注入（bindValue）
```

### 安全检查工具

`SecurityHelper::checkSecurity()` 在后台 Dashboard 运行，检测：
- `storage/` 目录是否有 `.htaccess` 保护
- `storage/site.db` 文件权限是否为 640/600/644
- 根目录 `.htaccess` 是否包含安全规则

### 密码安全

- 存储：`password_hash($pass, PASSWORD_DEFAULT)` → bcrypt
- 验证：`password_verify($input, $hash)`
- 默认账户：用户名 `admin`，首次安装后**必须立即修改密码**

## 13. 部署说明

### 运行环境需求

| 组件 | 最低要求 |
|------|---------|
| PHP | 8.1+ |
| PHP 扩展 | `sqlite3`、`mbstring`、`fileinfo`、`json` |
| Web 服务器 | Apache 2.4（需 `mod_rewrite`）或 Nginx |
| 磁盘空间 | ≥ 100 MB（媒体文件除外） |

### Apache 配置

确保 `.htaccess` 生效，需要：
```apache
AllowOverride All
```

### Nginx 配置（等效）

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}

# 禁止访问敏感目录
location ~* ^/(storage|\.git)/ { deny all; }
location ~* \.(db|sqlite|env|log)$ { deny all; }
```

### 初次部署步骤

1. 上传所有文件到 Web 根目录（或子目录）
2. 确认 Web 服务器对 `storage/`、`uploads/` 有写权限
3. 访问首页，系统自动执行所有迁移建库
4. 访问 `/admin/login`，使用 `admin` / `admin123` 登录
5. **立即在「设置 → 账户」修改默认密码**

### 子目录部署

无需配置，系统自动检测 `SCRIPT_NAME` 中的子目录路径并设置 `APP_BASE_PATH`。

**只需修改 `.htaccess` 中的 `RewriteBase`：**
```apache
RewriteBase /mysite/   # 对应子目录 /mysite/
```

## 14. 扩展指南

### 添加新前台页面

1. 在 `app/routes.php` 注册路由：
   ```php
   $router->add('GET', '/faq', [SiteController::class, 'faq']);
   ```

2. 在 `SiteController` 添加方法：
   ```php
   public function faq(): void {
       $this->renderSite('faq', [
           'seo' => ['title' => 'FAQ - ' . $this->siteData['site']['name']]
       ]);
   }
   ```

3. 创建模板文件 `themes/default/faq.php`

### 添加新数据库表

创建迁移文件，命名格式为当前时间戳 + 描述：
```
app/Migrations/20260412120000_create_faqs_table.php
```

内容参照现有迁移文件，使用匿名类格式。下次请求时自动执行。

### 添加新后台功能

1. 在 `app/routes.php` 注册后台路由
2. 在 `AdminController` 添加对应方法（含权限检查）
3. 在 `app/views/admin/` 创建视图文件
4. 在 `AuthManager::PERMISSION_MAP` 中添加权限映射（如需权限控制）

### 添加新区块

在 `themes/default/blocks.php` 中添加新条目，然后在对应模板中调用 `block()` 即可。用户保存时会写入 `storage/blocks/default.php`。
