# 网站模板开发指南

> 适用范围：`themes/` 目录下的网站主题开发  
> 目标读者：AI 大模型、模板开发者、后台区块配置维护者  
> 后台入口：`/admin/appearance/blocks`（外观区块 -> 模板区块）、`/admin/appearance/sliders`（外观区块 -> 轮播图）  
> 参考主题：`themes/default/`、`themes/hardware/`、`themes/furniture/`

---

## 1. 核心原则

开发主题时，必须把“可被客户改动的内容”放到 `blocks.php` 的模板区块中，模板文件只负责读取区块值并渲染页面。

必须遵守以下规则：

1. 产品展示区块必须使用 `product_picker` 产品选择器弹窗，不能用普通文本字段让用户手填产品 ID。
2. 图片、背景图、Logo、图标图片、证书图、视频、文件类媒体必须使用 `image` 或 `media` 字段，通过文件选择器弹窗选择，不能只提供普通文本 URL 字段。
3. 首页首屏轮播、Banner 轮播等轮播图区块必须使用 `/admin/appearance/sliders` 中已有的轮播图数据，并在模板区块中用 `select` 下拉框选择轮播图区块，默认使用第一个可用轮播图。
4. 区块要尽量可自定义：标题、描述、按钮文字、按钮链接、图片、视频、背景图、颜色、展示开关、数量、标签、统计数字、空状态文案都应暴露到后台。
5. 新主题前端样式优先使用 Tailwind CSS 类名，不要把大量视觉样式堆到 `style.css`。`style.css` 主要保留主题元数据和少量必须的补充样式。
6. 模板输出必须安全转义：文本和属性使用 `h()`，链接使用 `url()` 或经过验证的 URL，媒体路径使用 `asset_url()` 或主题内封装的媒体解析函数。

## 2. 主题目录结构

每个主题放在 `themes/{theme_name}/` 下。一个完整主题建议包含：

```text
themes/{theme_name}/
├── 404.php
├── about.php
├── blocks.php
├── case_detail.php
├── case_list.php
├── contact.php
├── footer.php
├── functions.php
├── header.php
├── home.php
├── list.php
├── page_detail.php
├── post_detail.php
├── post_list.php
├── product_detail.php
├── product_list.php
├── screenshot.jpg
├── style.css
└── thanks.php
```

文件职责：

| 文件 | 职责 |
| --- | --- |
| `blocks.php` | 定义后台模板区块、字段类型、默认值、下拉选项 |
| `functions.php` | 放主题级辅助函数，如产品卡片、媒体渲染、轮播读取 |
| `header.php` | 输出 HTML 头部、Tailwind 配置、导航栏 |
| `footer.php` | 输出页脚、浮动联系组件、全局脚本 |
| `home.php` | 首页区块编排 |
| `product_list.php` | 产品列表页 |
| `product_detail.php` | 产品详情页 |
| `post_*` / `case_*` / `page_detail.php` | 内容页模板 |
| `style.css` | 主题元数据，少量必要补充样式 |
| `screenshot.jpg` | 后台主题预览图 |

`style.css` 顶部必须包含主题信息：

```css
/*
Theme Name: Furniture
Theme URI: https://example.com
Author: Your Name
Description: B2B furniture website theme.
Version: 1.0.0
License: MIT
*/
```

## 3. 模板区块系统

### 3.1 `blocks.php` 基本结构

`blocks.php` 必须返回数组：

```php
<?php

return [
    'home_about' => [
        'label' => '首页 - 关于我们',
        'description' => '公司介绍、主图、按钮和统计数据。',
        'fields' => [
            'label' => ['type' => 'text', 'label' => '区块标签', 'default' => 'About Us'],
            'heading' => ['type' => 'text', 'label' => '标题', 'default' => 'Who We Are'],
            'text' => ['type' => 'textarea', 'label' => '描述', 'default' => 'Company introduction...'],
        ],
    ],
];
```

模板里通过 `block()` 读取：

```php
<h2><?= h(block('home_about', 'heading')) ?></h2>
<p><?= nl2br(h(block('home_about', 'text'))) ?></p>
```

也可以使用 `block_all()` 一次读取整个区块：

```php
<?php $about = block_all('home_about'); ?>
```

### 3.2 用户值和默认值

系统会合并两类值：

1. 主题默认值：`themes/{theme}/blocks.php`
2. 用户自定义值：`storage/blocks/{theme}.php`

读取优先级是“用户自定义值覆盖主题默认值”。如果后台清空字段，通常会回到默认值。

因此，`blocks.php` 的默认值必须可直接渲染页面，不能依赖用户先去后台配置。

### 3.3 区块命名规范

区块 key 使用 snake_case：

| 前缀 | 用途 |
| --- | --- |
| `brand_colors` | 全局品牌颜色 |
| `header` | 页头导航 |
| `footer` | 页脚 |
| `float_contact` | 浮动联系组件 |
| `home_*` | 首页区块 |
| `page_about_*` | 关于页区块 |
| `page_contact_*` | 联系页区块 |
| `product_*` | 产品列表、产品详情页通用配置 |
| `post_*` | 文章列表、文章详情页配置 |
| `case_*` | 案例列表、案例详情页配置 |

字段 key 也使用 snake_case，常用命名：

| 字段 key | 用途 |
| --- | --- |
| `label` | 区块小标签 |
| `heading` | 主标题 |
| `subheading` | 副标题 |
| `text` | 正文描述 |
| `btn_text` / `btn_url` | 按钮文字和链接 |
| `primary_text` / `primary_url` | 主按钮 |
| `secondary_text` / `secondary_url` | 次按钮 |
| `background_image` | 背景图 |
| `main_media` / `main_media_type` | 主媒体和媒体类型 |
| `product_ids` | 产品选择器保存的产品 ID 列表 |
| `slider_slug` | 轮播图 slug 下拉选择值 |
| `show_*` | 显示开关 |

## 4. 字段类型规范

后台模板区块当前常用字段类型如下：

| 类型 | 后台控件 | 使用场景 |
| --- | --- | --- |
| `text` | 单行输入框 | 短文本、按钮文字、链接、标签、数字 |
| `textarea` | 多行输入框 | 描述、长文案、分行文案 |
| `color` | 颜色选择器 | 品牌色、背景色、文字色、边框色 |
| `image` | 图片输入框 + 图片选择器弹窗 | 图片、背景图、Logo、图标图片 |
| `media` | 媒体输入框 + 文件选择器弹窗 | 图片或视频，支持自动保存媒体类型 |
| `product_picker` | 产品选择器弹窗 | 所有产品展示、精选产品、热销产品、相关产品 |
| `icon` | 图标字段 | Font Awesome 图标类名 |
| `select` | 下拉框 | 轮播图选择、显示开关、布局样式、数量模式 |
| `hidden` | 隐藏字段 | 兼容旧字段、媒体类型、备用值 |

字段设计要求：

1. 后台用户需要选择图片时，用 `image`。
2. 后台用户可能选择图片或视频时，用 `media`，并配置 `media_type_key`。
3. 后台用户需要选产品时，用 `product_picker`。
4. 后台用户需要选择轮播图时，用 `select`，选项来自 `App\Models\Slider`。
5. 不要让用户在普通 `text` 字段里填写图片地址、视频地址、产品 ID、轮播 slug，除非这是纯技术备用字段且设为 `hidden`。

## 5. 产品展示区块规范

### 5.1 强制使用产品选择器弹窗

任何产品展示区块都必须使用 `product_picker`：

```php
'home_featured' => [
    'label' => '首页 - 热销产品',
    'description' => '使用商品选择弹窗勾选商品；未勾选时默认展示最新上架商品。',
    'fields' => [
        'label' => ['type' => 'text', 'label' => '区块小标题', 'default' => 'Hot Products'],
        'heading' => ['type' => 'text', 'label' => '标题', 'default' => 'Hot Products'],
        'text' => ['type' => 'textarea', 'label' => '描述', 'default' => 'Popular products for global buyers.'],
        'more_text' => ['type' => 'text', 'label' => '更多按钮文字', 'default' => 'More Products'],
        'more_url' => ['type' => 'text', 'label' => '更多按钮链接', 'default' => '/products'],
        'product_ids' => [
            'type' => 'product_picker',
            'label' => '勾选商品列表',
            'default' => '',
            'limit' => 8,
            'status' => 'active',
        ],
    ],
],
```

模板读取方式：

```php
<?php
$selectedProductIds = trim(block('home_featured', 'product_ids'));
$selectedProducts = $selectedProductIds !== '' ? get_products_by_ids($selectedProductIds, true) : [];
$featuredProducts = !empty($selectedProducts)
    ? $selectedProducts
    : get_products(['limit' => 8, 'status' => 'active']);
?>
```

### 5.2 默认行为

产品区块默认规则：

1. 用户已勾选产品：按用户勾选顺序展示。
2. 用户未勾选产品：展示最新 active 产品。
3. 如果仍然没有产品：显示可配置的空状态文案或隐藏产品列表区域。

示例：

```php
<?php if (!empty($featuredProducts)): ?>
    <div class="grid grid-cols-1 gap-6 sm:grid-cols-2 lg:grid-cols-4">
        <?php foreach ($featuredProducts as $product): ?>
            <?php render_product_card($product); ?>
        <?php endforeach; ?>
    </div>
<?php else: ?>
    <p class="text-sm text-slate-500"><?= h(block('home_featured', 'empty_text', 'No products yet.')) ?></p>
<?php endif; ?>
```

### 5.3 禁止写法

产品展示区块禁止这样设计：

```php
'item1_title' => ['type' => 'text', 'label' => '产品 1 标题', 'default' => 'Product A'],
'item1_image' => ['type' => 'image', 'label' => '产品 1 图片', 'default' => '/uploads/a.jpg'],
'item1_url' => ['type' => 'text', 'label' => '产品 1 链接', 'default' => '/product/a'],
```

原因：这会让产品数据和真实商品库脱节，后台无法复用产品选择弹窗、库存、分类、详情页链接、询盘流程。

如果为了旧主题兼容必须保留静态产品字段，只能设置为 `hidden` 作为兜底，不应作为后台主要配置项。

## 6. 文件、图片、视频字段规范

### 6.1 图片使用 `image`

适用于单纯图片：

```php
'background_image' => [
    'type' => 'image',
    'label' => '背景图片',
    'default' => '/uploads/default-bg.jpg',
],
```

模板输出：

```php
<?php $backgroundImage = block('home_why', 'background_image'); ?>
<?php if ($backgroundImage !== ''): ?>
    <img src="<?= h(asset_url($backgroundImage)) ?>" alt="" class="h-full w-full object-cover">
<?php endif; ?>
```

### 6.2 图片或视频使用 `media`

适用于主视觉、工厂媒体、关于我们主图、产品介绍视频等：

```php
'main_media_type' => [
    'type' => 'hidden',
    'label' => '主媒体类型',
    'default' => 'image',
],
'main_media' => [
    'type' => 'media',
    'label' => '主媒体',
    'default' => '/uploads/default-factory.jpg',
    'media_type_key' => 'main_media_type',
    'legacy_image_key' => 'main_image',
    'legacy_video_key' => 'main_video',
    'allowed' => 'all',
],
```

模板输出：

```php
<?php
$mediaUrl = block('home_about', 'main_media');
$mediaType = block('home_about', 'main_media_type', 'image');
?>

<?php if ($mediaUrl !== ''): ?>
    <?php if ($mediaType === 'video'): ?>
        <video class="h-full w-full object-cover" autoplay muted loop playsinline>
            <source src="<?= h(asset_url($mediaUrl)) ?>">
        </video>
    <?php else: ?>
        <img src="<?= h(asset_url($mediaUrl)) ?>" alt="<?= h(block('home_about', 'heading')) ?>" class="h-full w-full object-cover">
    <?php endif; ?>
<?php endif; ?>
```

### 6.3 文件选择器使用要求

凡是后台需要选文件、图片、视频的地方，都必须走文件选择器弹窗：

1. 图片：`type => image`
2. 图片或视频：`type => media`
3. 只允许图片：`allowed => image`
4. 只允许视频：`allowed => video`
5. 图片和视频都允许：`allowed => all`

禁止把图片、视频、文件设计成普通 `text` 字段让用户手动粘贴路径。

## 7. 轮播图区块规范

### 7.1 轮播图数据来源

轮播图片、标题、描述、按钮文字和链接应在 `/admin/appearance/sliders` 中维护。

模板区块只负责选择“使用哪一个轮播图区块”，不负责逐张维护轮播图。

### 7.2 `blocks.php` 中生成轮播图下拉选项

推荐在 `blocks.php` 顶部读取已有 active 轮播图：

```php
<?php

$sliderOptions = [];
try {
    if (class_exists(\App\Models\Slider::class)) {
        $sliderModel = new \App\Models\Slider();
        foreach ($sliderModel->getAll() as $slider) {
            if (($slider['status'] ?? 'active') !== 'active') {
                continue;
            }

            $slug = trim((string)($slider['slug'] ?? ''));
            if ($slug === '') {
                continue;
            }

            $name = trim((string)($slider['name'] ?? ''));
            $sliderOptions[$slug] = ($name !== '' ? $name : $slug) . ' (' . $slug . ')';
        }
    }
} catch (\Throwable $e) {
    $sliderOptions = [];
}

if (empty($sliderOptions)) {
    $sliderOptions = ['home-hero' => '首页轮播图 (home-hero)'];
}

$defaultHeroSlider = (string)array_key_first($sliderOptions);
```

区块字段：

```php
'home_hero' => [
    'label' => '首页 - 首屏轮播',
    'description' => '图片、标题、描述、链接来自 外观区块 -> 轮播图。',
    'fields' => [
        'slider_slug' => [
            'type' => 'select',
            'label' => '选择轮播图区块',
            'default' => $defaultHeroSlider,
            'options' => $sliderOptions,
        ],
        'autoplay' => [
            'type' => 'select',
            'label' => '自动轮播',
            'default' => 'yes',
            'options' => ['yes' => '启用', 'no' => '关闭'],
        ],
        'show_overlay' => [
            'type' => 'select',
            'label' => '显示文字层',
            'default' => 'yes',
            'options' => ['yes' => '显示', 'no' => '隐藏'],
        ],
    ],
],
```

### 7.3 模板读取轮播图

在 `functions.php` 中提供辅助函数：

```php
if (!function_exists('get_slider_items')) {
    function get_slider_items(string $sliderSlug): array {
        try {
            $sliderModel = new \App\Models\Slider();
            $slider = $sliderModel->getBySlugWithItems($sliderSlug);

            if (!$slider || ($slider['status'] ?? '') !== 'active') {
                return [];
            }

            $items = [];
            foreach (($slider['items'] ?? []) as $item) {
                $items[] = [
                    'id' => $item['id'],
                    'title' => $item['title'],
                    'subtitle' => $item['subtitle'],
                    'image' => $item['image'],
                    'url' => $item['link_url'] ?: '#',
                    'link_text' => $item['link_text'] ?: 'View Details',
                ];
            }

            return $items;
        } catch (\Throwable $e) {
            return [];
        }
    }
}
```

在 `home.php` 中读取：

```php
<?php
$heroSliderSlug = trim(block('home_hero', 'slider_slug'));
$heroSlides = $heroSliderSlug !== '' ? get_slider_items($heroSliderSlug) : [];
?>
```

如果没有轮播图数据，可以使用隐藏备用图兜底，但后台可见配置必须仍然是轮播图下拉选择。

### 7.4 禁止写法

禁止把首页首屏轮播设计成这些后台字段：

```php
'slide1_image' => ['type' => 'image', 'label' => '轮播图 1', 'default' => '...'],
'slide2_image' => ['type' => 'image', 'label' => '轮播图 2', 'default' => '...'],
'slide3_image' => ['type' => 'image', 'label' => '轮播图 3', 'default' => '...'],
```

如果旧主题需要保留这些字段，只能作为 `hidden` fallback。实际维护入口必须是 `/admin/appearance/sliders`。

## 8. 可自定义内容覆盖范围

生成主题时，应尽量让后台可配置以下内容：

### 8.1 全局

1. 品牌主色、深色、强调色、文字色、边框色、背景色。
2. Header 的 Logo、备用品牌文字、电话、邮箱、搜索框提示、CTA 按钮、社媒链接。
3. Footer 的公司简介、栏目标题、版权文字、社媒链接、联系信息标题。
4. 浮动联系组件的开关、按钮文字、WhatsApp、邮箱、电话、表单入口。

### 8.2 首页

1. 首屏轮播：轮播图下拉、自动轮播、文字层开关。
2. 产品区块：标题、描述、更多按钮、产品选择器、空状态文案。
3. 关于我们：标题、描述、按钮、主媒体、统计数字、角标。
4. 优势区：背景图、标题、描述、图标、卡片标题、卡片文案。
5. 应用场景：标题、描述、每个场景图片、标题、文案。
6. 工厂/能力展示：图片或视频、统计数据、证书、流程说明。
7. FAQ：问题、答案、标签。
8. CTA：背景图、标题、描述、按钮、联系信息。

### 8.3 产品页

1. 列表页 Banner 标题、描述、背景图。
2. 筛选区标题、分类标题、搜索提示。
3. 产品卡片按钮文字，如 `View Details`、`Get Quote`。
4. 产品详情页的询盘按钮、参数标题、描述标题、相关产品标题。
5. 相关产品建议使用 `product_picker`，未选择时按分类或最新产品兜底。

### 8.4 内容页

1. 关于页：Banner、公司简介、团队/工厂图片、视频、统计数据。
2. 联系页：Banner、表单标题、联系方式标题、地图/背景图。
3. 文章/案例列表页：Banner、卡片按钮、空状态文案。
4. 文章/案例详情页：返回按钮、相关内容标题、CTA。

## 9. Tailwind CSS 规范

新主题应优先使用 Tailwind CSS 类名：

```php
<section class="bg-white py-16 sm:py-20 lg:py-24">
    <div class="mx-auto max-w-7xl px-4 sm:px-6 lg:px-8">
        <div class="grid gap-8 lg:grid-cols-2 lg:items-center">
            ...
        </div>
    </div>
</section>
```

`style.css` 中只放：

1. 主题元数据。
2. Tailwind 无法方便表达的极少量补充样式。
3. 第三方组件必要修正。

不要把整站按钮、卡片、布局、响应式样式全部写进 `style.css`。

颜色建议通过 `brand_colors` 注入 Tailwind 配置或 CSS 变量：

```php
<?php
$brandPrimary = block('brand_colors', 'primary', '#2563eb');
$brandDark = block('brand_colors', 'primary_dark', '#1e3a8a');
?>
<script>
tailwind.config = {
    theme: {
        extend: {
            colors: {
                brand: {
                    500: '<?= h($brandPrimary) ?>',
                    900: '<?= h($brandDark) ?>'
                }
            }
        }
    }
}
</script>
```

## 10. 模板代码规范

### 10.1 安全输出

必须使用：

```php
<?= h($text) ?>
<?= nl2br(h($textarea)) ?>
<a href="<?= h(url('/products')) ?>">
<img src="<?= h(asset_url($image)) ?>" alt="<?= h($title) ?>">
```

避免：

```php
<?= $text ?>
<img src="<?= $image ?>">
```

### 10.2 URL 和媒体路径

1. 站内链接使用 `url('/path')`。
2. 上传文件或相对路径使用 `asset_url($path)`。
3. 远程 URL 可以原样保存，但输出时仍要 `h()`。
4. 产品详情链接使用产品真实 slug：`url('/product/' . $product['slug'])`。

### 10.3 主题辅助函数

重复逻辑应放在 `functions.php`：

1. `render_product_card(array $product): void`
2. `render_media(string $url, string $type, string $class = ''): void`
3. `get_slider_items(string $slug): array`
4. `theme_image_url(string $path): string`
5. `theme_format_price(array $product): string`

不要在多个模板里复制大量产品卡片、媒体类型判断、轮播读取逻辑。

## 12. 页面模板设计要点

### 12.1 `home.php`

首页应是多个区块的组合，每个区块都从 `blocks.php` 读取内容。

推荐顺序：

1. `home_hero`：轮播图下拉选择。
2. `home_featured`：产品选择器。
3. `home_about`：文字、按钮、图片/视频。
4. `home_why`：优势卡片和背景图。
5. `home_applications`：应用场景图片。
6. `home_process`：流程。
7. `home_faq`：FAQ。
8. `home_cta`：行动召唤。

### 12.2 `product_list.php`

产品列表页必须使用真实产品数据，不要用区块伪造产品卡片。

区块只配置页面文案、Banner、按钮文字、空状态、筛选提示等。

### 12.3 `product_detail.php`

产品详情页应使用产品数据库字段作为主体：

1. 产品标题、图片、摘要、描述、参数、分类、SEO 来自产品数据。
2. 区块配置只控制页面辅助文案和样式，例如询盘按钮、参数标题、相关产品标题。
3. 相关产品如果需要人工控制，使用 `product_picker`；否则按当前分类或最新产品兜底。
4. 产品详情图集来自产品数据或媒体字段，不要在模板区块里逐张配置某个产品的详情图。

### 12.4 `header.php` 和 `footer.php`

Header 和 Footer 应优先读取全局站点设置，如 `$site['logo']`、`$site['company_phone']`、`$site['company_email']`。如果全局设置为空，再使用区块默认值。

示例：

```php
<?php
$phone = trim((string)($site['company_phone'] ?? ''));
if ($phone === '') {
    $phone = block('header', 'top_phone');
}
?>
```

## 13. AI 生成主题时的工作流程

AI 大模型生成或修改主题时，应按以下流程执行：

1. 阅读 `themes/default/blocks.php`、`themes/hardware/blocks.php`、目标主题 `blocks.php`，确认项目实际字段类型。
2. 阅读 `app/views/admin/blocks/index.php`，确认后台字段支持能力。
3. 阅读 `assets/admin/base.js`，确认产品选择器和文件选择器弹窗能力。
4. 设计 `blocks.php`，先确定每个区块需要暴露哪些可配置项。
5. 所有产品展示统一使用 `product_picker`。
6. 所有图片、视频、文件统一使用 `image` 或 `media`。
7. 所有轮播图统一使用 `select` 选择 `/admin/appearance/sliders` 里的轮播图区块。
8. 编写模板文件，使用 Tailwind 类名实现视觉效果。
9. 抽取重复渲染逻辑到 `functions.php`。
10. 运行 PHP 语法检查。
11. 在前台和后台模板区块页检查字段是否可编辑、可选择、可预览。

## 14. 验收清单

开发完成后必须检查：

### 14.1 文件完整性

1. 主题目录包含必须模板文件。
2. `style.css` 有主题元数据。
3. `screenshot.jpg` 存在。
4. `blocks.php` 可以正常 `include` 并返回数组。

### 14.2 模板区块

1. 产品区块使用 `product_picker`。
2. 产品未选择时有最新产品兜底。
3. 图片使用 `image` 字段。
4. 图片/视频混合媒体使用 `media` 字段。
5. `media` 字段配置了 `media_type_key`。
6. 轮播图区块使用 `select` 选择已有轮播图。
7. 轮播图默认值是第一个可用轮播图。
8. 区块标题、描述、按钮、图片、背景图、颜色、开关都尽量可配置。
9. 不存在让用户手填产品 ID 的普通文本字段。
10. 不存在让用户手填图片/视频路径的主要配置字段。

### 14.3 前端渲染

1. 所有动态文本使用 `h()` 转义。
2. 多行文本使用 `nl2br(h(...))`。
3. 媒体路径使用 `asset_url()` 或主题媒体辅助函数。
4. 产品链接使用真实 slug。
5. 空数据状态不报错。
6. 移动端、平板、桌面布局不重叠、不溢出。
7. Tailwind 类名承担主要视觉样式。
8. `style.css` 没有大量自定义页面样式。

### 14.4 命令检查

PHP 语法检查：

```bash
find themes/{theme_name} -name '*.php' -print0 | xargs -0 -n1 php -l
```

检查产品选择器、媒体选择器、轮播下拉是否存在：

```bash
rg "product_picker|media_type_key|slider_slug|type' => 'select'" themes/{theme_name}/blocks.php
```

检查是否有可疑的静态产品字段：

```bash
rg "item[0-9]+_(title|image|url)" themes/{theme_name}/blocks.php
```

如果命中静态产品字段，需要确认它们是否为 `hidden` fallback；否则应改为 `product_picker`。

## 15. 禁止事项

1. 禁止产品展示区块使用 `item1_title`、`item1_image`、`item1_url` 作为主要配置。
2. 禁止让后台用户用普通文本框填写产品 ID。
3. 禁止让后台用户用普通文本框填写图片、视频、文件路径作为主要配置。
4. 禁止首页首屏轮播逐张写在模板区块里，必须使用轮播图管理和下拉选择。
5. 禁止在模板中硬编码大量前台文案、图片、按钮和链接。
6. 禁止直接输出未转义的后台配置值。
7. 禁止把新主题的大部分视觉样式写进 `style.css`。
8. 禁止为了实现主题效果修改系统核心表结构，除非任务明确要求后端功能扩展。
9. 禁止删除或覆盖用户已有的后台区块配置文件。
10. 禁止在主题代码里执行写数据库、删文件、重置配置等破坏性操作。

## 17. 主题质量标准

一套合格主题应满足：

1. 后台可维护：客户不需要改代码就能换文字、换图、换视频、换产品、换轮播。
2. 数据真实：产品、文章、案例、轮播图来自系统数据，不用模板假数据冒充。
3. 样式现代：使用 Tailwind CSS 实现响应式、清晰层级、稳定间距。
4. 代码稳定：无 PHP 语法错误，空数据不报错。
5. 安全可靠：输出转义，链接和媒体路径处理正确。
6. 易复用：重复 UI 抽成主题辅助函数，后续主题能参考。
7. 少侵入：主题目录内完成主题能力，不随意修改系统核心。

本指南是后续 AI 生成网站主题的硬性设计规范。凡涉及产品、媒体、轮播的模板区块，都应优先遵守对应选择器规则。
